# Dropbox 構造定義

## 概要

Dropbox は静的保管庫として機能します。
ファイルを「整理する場所」ではなく「流れ着く場所」として設計します。

---

## フォルダ構造

```
00_DocumentVault/
├── 01_Action/
├── 02_Archive/
├── 03_Temporary/
└── 99_Imported/
```

---

## 各フォルダの役割

### 01_Action/
- **用途**: 期限・アクションが残っている書類
- **格納基準**: Notion に登録済みで、対応完了前のもの
- **整理タイミング**: アクション完了後に 02_Archive/ へ移動

### 02_Archive/
- **用途**: 対応完了・参照用として保管する書類
- **格納基準**: 処理済みで長期参照が必要なもの
- **整理タイミング**: 保存期限到来時に削除

### 03_Temporary/
- **用途**: 一時的な保管・作業中ファイル
- **格納基準**: 月内で処理予定だが分類未確定のもの
- **整理タイミング**: 月末に必ず 01_Action/ か 02_Archive/ へ振り分ける

### 99_Imported/
- **用途**: ローカルの export/ から移動してきたファイルの一次受け
- **格納基準**: 分類前の import 直後ファイル
- **整理タイミング**: 月次処理時に各フォルダへ振り分ける

---

## 運用ルール

1. ローカルの `export/` から Dropbox へ移動する際は、まず `99_Imported/` へ入れる
2. 月次処理時に `99_Imported/` の中身を各フォルダへ振り分ける
3. `03_Temporary/` は月を跨いで放置しない
4. ファイル名は `YYYYMMDD_カテゴリ_相手先_概要.pdf` 形式を維持する

---

## ファイル命名規則

```
YYYYMMDD_カテゴリ_相手先_概要.pdf
```

### カテゴリ例

| カテゴリ | 説明 |
|---|---|
| CONTRACT | 契約書 |
| INVOICE | 請求書・領収書 |
| RECEIPT | 領収書（小額） |
| REPORT | 報告書・議事録 |
| NOTICE | 通知書・案内 |
| PERMIT | 許認可・資格証明 |
| OTHER | 上記以外 |

### 例

```
20260501_CONTRACT_株式会社A_業務委託契約書.pdf
20260510_INVOICE_B社_2026年4月分請求書.pdf
20260515_RECEIPT_コンビニ_交通費領収書.pdf
```

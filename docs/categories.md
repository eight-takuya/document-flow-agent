# Category 一覧

ファイル命名規約の `Category` フィールドで使用できる分類の一覧です。

命名規約の詳細は `docs/naming-convention.md` を参照してください。

---

## 一覧

| Category | 説明 |
|---|---|
| `Action` | 期限・対応が必要な書類。Notion 登録を優先する |
| `Tax` | 税金関連。住民税・所得税・固定資産税・確定申告など |
| `Insurance` | 保険関連。生命保険・自動車保険・火災保険など |
| `Medical` | 医療・健康関連。診察券・領収書・検査結果・処方箋など |
| `School` | 教育関連。学校からのお知らせ・請求書・授業料・成績通知など |
| `Contract` | 契約書・同意書・見積書・利用規約への署名など |
| `Work` | 業務関連。業務完了通知書・発注書・納品書・請求書（仕事）など |
| `Expense` | 経費・支出記録。会食・交通費・備品購入など |
| `Receipt` | 小額領収書・レシート類（Expense より粒度が小さいもの） |
| `Utility` | 公共料金。電気・ガス・水道・通信費など |
| `Government` | 行政・公的機関からの書類。通知書・証明書・許可証・法人登記など |
| `Family` | 家族関連。育児・介護・家族全体に関わる書類 |
| `Asset` | 資産関連。不動産・車・投資・口座情報など |
| `Other` | 上記に該当しないもの。分類に迷ったら一旦ここへ |

## 実ファイルからの分類ガイド

inbox に実際に現れやすいファイルと推奨 Category の対応例です。

| ファイル種別 | 推奨 Category | 補足 |
|---|---|---|
| 成績通知 | `School` | - |
| 業務完了通知書 | `Work` | 相手先・案件名を Counterparty に入れると管理しやすい |
| 御見積書 | `Contract` | 正式契約前の段階でも Contract で可 |
| 履歴事項全部証明書 | `Government` | 法人登記系は Government |
| 適用事業所所在名称変更通知書 | `Government` | 社会保険・行政への届出も Government |
| ETCカード関連お知らせ | `Government` | カード会社 → Other でも可 |
| お礼状 | `Other` | - |
| チラシ・販促物・無料回収案内 | — | 廃棄推奨。archive/export 不要 |
| 交通反則通告制度の案内 | `Government` | 法律・制度の情報提供 |
| 自転車関連の行政通知 | `Government` | - |

---

## 廃棄候補の判断基準

以下に該当する場合は保管不要として廃棄を検討してください。

- チラシ・DM・販促物
- 内容のないお知らせ（クレジットカード会社の軽微なお知らせ等）
- 既に対応済みで参照の必要がないもの
- Notion にも Dropbox にも保管する理由がないもの

廃棄した場合はログに記録し、inbox から削除する。archive/export には移動しない。

## 【TAG】形式のファイル名について

`【B8E】` のようなブラケットタグが付いたファイルは、**会社（法人）関連書類**を示します。

リネーム時は、タグを `Counterparty` フィールドに移動します。

```
# 変換前（inbox 内）
20260407【B8E】履歴事項全部証明書.pdf

# 変換後（archive/export 内）
20260407-Government-履歴事項全部証明書-B8E.pdf
```

## `Work` と `Contract` の使い分け

| 状況 | Category |
|---|---|
| 仕事の完了報告・進捗報告 | `Work` |
| 見積書・発注書・納品書 | `Contract` |
| 正式な契約書 | `Contract` |
| 業務委託の完了通知 | `Work` |

---

## 運用上の注意（Claude Cowork 向け）

- `Action` は期限・アクションが明確な場合のみ使う。対応完了後は別 Category に変更する
- `Receipt` と `Expense` の使い分け: 金額が小さくメモ代わりのものは `Receipt`、経費申請・記録として残すものは `Expense`
- `Other` が溜まってきたら月次で見直し、新 Category 追加の検討材料にする
- ファイル名が文字化けしているものは、PDF を開いて内容を確認してから Category を決定する
